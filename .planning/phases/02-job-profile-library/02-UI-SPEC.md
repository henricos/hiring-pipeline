## D-03 Decision: Section Delimiter — Heading over Card

**Decision: Use section headings with vertical whitespace, not cards.**

Rationale: The form has ~6 sections across a long single page. If each section were wrapped in a `surface-container-low` card on a `surface` background, the visual weight stacks and creates a card-inside-card nesting that conflicts with the surface hierarchy rule: cards are reserved for list-row items (profile list) and modal-adjacent contexts, not grouping containers inside a workspace.

The correct application of the No-Line Rule here is: the entire form workspace sits on `surface-container-lowest` (#ffffff) — that single tonal lift from `surface` (#f8f9fa) already separates the form from the page. Within the form, section headings (`title-md`, 18px, weight 500, color `on-surface`) separated by `xl` (32px) top margin provide visual grouping without adding another surface layer.

This follows DESIGN.md §5 Cards rule: "Use 1.5rem to 2rem of vertical whitespace to separate entries."

**Implementation contract:**
- Form container: `bg-surface-container-lowest`, `px-8 py-8`, `rounded-md`, `shadow-ambient`
- Section title: `text-title-md font-medium text-on-surface`, `mt-8 mb-4`
- No wrapping card per section
- A 1px `outline-variant` at 15% opacity (`ghost border fallback`) may be used as a subtle visual rule between sections ONLY if user testing reveals ambiguity — not by default

## Form Layout Contract

### /profiles — List Page

- Container: full-width within shell workspace (`surface-container-lowest`)
- Page header: `headline-sm` page title "Perfis de Vaga" + "Novo perfil" button (right-aligned, primary CTA)
- List: unstyled rows, each row on `surface-container-lowest`; separation by `md` (16px) vertical whitespace — no dividers (No-Line Rule)
- Each row contains:
  - **Cargo** (title): `body-md`, weight 500, `on-surface`
  - **Cargo sugerido** (subtitle): `body-md`, weight 400, `on-surface/70`, shown below title
  - **Última atualização**: `label-sm`, all-caps, `on-surface/70`, right column
  - **Actions**: icon-only buttons — edit (pencil, `Users` → use `Pencil` from lucide), delete (trash, `Trash2` from lucide); 40px minimum touch target; color `on-surface/50`, hover `on-surface`
- Sort: most-recently-updated first (D-05) — no sort UI control exposed
- Empty state: see Copywriting section

### /profiles/new and /profiles/[id]/edit — Form Pages

- Page header: `headline-sm` — "Novo perfil" or "Editar perfil"
- Form container: `bg-surface-container-lowest rounded-md shadow-ambient px-8 py-8 max-w-3xl mx-auto`
- Section structure: heading (`title-md`) + form fields below, separated by `xl` top margin
- Sections (in order):
  1. **Identificação** — título do cargo, cargo sugerido para anúncio
  2. **Requisitos do Candidato** — experiência, escolaridade (+ curso), pós-graduação (+ curso), certificações (+ quais), idiomas (inglês, espanhol, outro idioma)
  3. **Conteúdo Descritivo** — 5 textareas (responsabilidades, requisitos/qualificações, comportamentais, principais desafios, informações complementares)
  4. **Infraestrutura** — sistemas necessários, pastas de rede (both optional)
  5. **Observações Internas** — textarea, not published externally; hint text makes this explicit
- Save button: bottom of form, right-aligned, primary CTA with gradient; label "Salvar perfil"
- Conditional sub-fields: "Curso" field appears inline below escolaridade/pós-graduação select when relevant option is selected; "Quais" field appears inline below certificações when "Desejável" or "Sim" is selected

### Select Field Values

| Field | Options |
|-------|---------|
| Tempo de experiência | < 1 ano / 1–3 anos / 3–5 anos / 5–10 anos / > 10 anos |
| Nível de escolaridade | Ensino médio / Superior cursando / Superior completo |
| Pós-graduação | Não exigido / Desejável / Necessário |
| Certificações | Não / Desejável / Sim |
| Inglês | Não exigido / Básico / Intermediário / Avançado / Fluente |
| Espanhol | Não exigido / Básico / Intermediário / Avançado / Fluente |

"Outro idioma" uses two inline fields: text input (nome do idioma) + select (nível, same options as Inglês/Espanhol).

## Interaction Contracts

| Interaction | Contract |
|-------------|----------|
| Click profile row | Navigate to `/profiles/[id]/edit` — entire row is clickable (D-08) |
| Click "Novo perfil" button | Navigate to `/profiles/new` (D-09) |
| Click "Salvar perfil" | Submit form via Server Action; show loading state on button (spinner replaces icon); on success navigate to `/profiles`; on error show inline error below form |
| Click delete icon | Open shadcn `AlertDialog` (D-10); does NOT navigate away |
| AlertDialog confirm | Execute delete Server Action; navigate to `/profiles` on success |
| AlertDialog cancel | Close dialog; return to list with no change |
| Conditional sub-fields | Appear immediately on select change (no page navigation); animate with `transition-all duration-150` |
| Save with validation error | Highlight errored fields with `destructive` bottom bar; scroll to first error; do not navigate away |

## Accessibility Contracts

- All form fields paired with `<Label htmlFor>` — no placeholder-only labeling
- Select elements use shadcn Select (Radix UI) — keyboard navigable
- AlertDialog traps focus inside dialog while open (Radix behavior)
- Delete icon button: `aria-label="Excluir perfil [nome]"` with profile name interpolated
- Edit icon button: `aria-label="Editar perfil [nome]"` with profile name interpolated
- Error messages: `role="alert"` or associated via `aria-describedby` on the field
- Color contrast: `on-surface` (#2b3437) on `surface-container-lowest` (#ffffff) — WCAG AA passes
