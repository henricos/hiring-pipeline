Executar os seguintes comandos para levantar o estado atual:

**1. Listar arquivos legados em DATA_PATH/research/ (flat):**
```bash
source .env.local && ls "$DATA_PATH/research/"*.json 2>/dev/null || echo "Nenhum arquivo legado encontrado"
```

**2. Listar perfis disponíveis com ID completo:**
```bash
source .env.local && node -e '
const fs = require("fs");
const path = require("path");
const profilesDir = path.join(process.env.DATA_PATH, "profiles");
const files = fs.readdirSync(profilesDir).filter(f => f.endsWith(".json"));
files.forEach(f => {
  const p = JSON.parse(fs.readFileSync(path.join(profilesDir, f), "utf8"));
  console.log(p.id + " | " + p.title + " | " + p.experienceLevel);
});
'
```

**3. Para cada arquivo legado que o gestor quiser migrar:**

Substituir `{PROFILE_ID}` pelo UUID completo do perfil correspondente e `{ARQUIVO_LEGADO}`
pelo nome do arquivo (ex: `dev-frontend-senior-2026-04-25`):

```bash
source .env.local && \
  mkdir -p "$DATA_PATH/research/{PROFILE_ID}" && \
  mv "$DATA_PATH/research/{ARQUIVO_LEGADO}-vagas.json" "$DATA_PATH/research/{PROFILE_ID}/2026-04-25-vagas.json" && \
  mv "$DATA_PATH/research/{ARQUIVO_LEGADO}-resumo.json" "$DATA_PATH/research/{PROFILE_ID}/2026-04-25-resumo.json" && \
  echo "Migrado com sucesso"
```

Nota: O nome do arquivo de destino deve seguir o padrao `{YYYY-MM-DD}-vagas.json` —
extrair a data do nome do arquivo legado.

**4. Verificar resultado da migracao:**
```bash
source .env.local && find "$DATA_PATH/research" -name "*-resumo.json" | sort
```
