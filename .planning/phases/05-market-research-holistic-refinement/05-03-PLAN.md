**7 Steps completos:**

**Step 1 — Coletar escopo conversacional:**
Perguntar ao gestor (em conversa natural, não formulário rígido):
- Cargo/função a pesquisar (ex: "Engenheiro Sênior de Software", "Staff Engineer")
- Localização (default: São Paulo)
- Senioridade (Pleno / Sênior / Staff / Principal / Arquiteto)
- Stacks-chave de interesse (ex: Java + Python + TypeScript)
- Indústria/setor (default: Tecnologia)
- **Profundidade da pesquisa** (apresentar as 3 opções):
  - `enxuta` — 5-10 vagas, ~5 min
  - `média` — 15-25 vagas, ~15 min (padrão recomendado)
  - `profunda` — 30-50 vagas, ~30 min+
- **Porte de empresa** (default: médias+):
  - `todas` — incluir startups e pequenas empresas
  - `médias+` — filtrar para médias e grandes (default)
  - `grandes+` — apenas empresas de grande porte

Gerar slug sanitizado: converter cargo+local+data em kebab-case, aceitar apenas `[a-z0-9-]+`.
Exemplos: "senior-pd-java-python-ts-sp", "staff-engineer-sp"
REJEITAR slugs com `..`, `/`, `\`, espaços — gerar novo automaticamente.

Após coletar, exibir resumo e aguardar confirmação:
```
Escopo definido:
  Cargo: {role} ({seniority})
  Local: {location}
  Stacks: {stack[].join(', ')}
  Profundidade: {depth}
  Porte: {companySize}
  Slug: {slug}-{YYYY-MM-DD}

Confirmar e iniciar pesquisa? (S/N)
```

**Step 2 — Detectar sessões autenticadas e executar queries WebSearch:**

Primeiro, verificar sessões disponíveis:
```bash
ls $DATA_PATH/sessions/ 2>/dev/null
```
Para cada arquivo `{portal}-session.json` encontrado, registrar que sessão está disponível para aquele portal. NÃO logar o conteúdo do arquivo — apenas confirmar existência.

Exibir:
```
Sessões autenticadas detectadas: {lista de portais com sessão} (ou "nenhuma")
```

Em seguida, executar queries WebSearch nos portais aprovados (lista real do 05-01-PORTALS.md).
Usar queries documentadas no 05-01-PORTALS.md — tanto PT quanto EN conforme recomendação.

Para cada portal aprovado:
- Executar query WebSearch com o cargo/senioridade/local do escopo
- Registrar cada resultado em `sources[]` com: portal, query, url, status

**Step 3 — WebFetch das descrições mais relevantes:**

Selecionar URLs para WebFetch conforme profundidade escolhida:
- `enxuta`: top 5-10 resultados
- `média`: top 15-25 resultados
- `profunda`: top 30-50 resultados

Priorização quando há mais candidatos do que a profundidade permite:
- Priorizar vagas de portais com WebFetch OK (evitar portais que retornam 403 consistentemente)
- Priorizar vagas com snippets mais ricos (mais texto sobre requisitos/stack)
- Priorizar empresas que parecem médias/grandes (quando filtro é médias+)

Para portais com sessão autenticada disponível: usar a sessão para o WebFetch daquele portal.
Se WebFetch falhar (403, timeout): registrar como `status: "unavailable"` em `sources[]` e prosseguir — sem retry agressivo.

**Step 4 — Extrair e estruturar por vaga + aplicar filtro de porte:**

Para cada vaga com fetch bem-sucedido, extrair:
- `title` — título exato da vaga
- `company` — nome da empresa
- `companySize` — estimativa: heurística baseada em nome + contexto do snippet:
  - Multinacional conhecida / grande empresa BR → "grande"
  - Empresa com produto estabelecido, sem sinais de startup → "média"
  - Startup, poucos funcionários visíveis, sem histórico → "pequena"
  - Não identificável → "desconhecido"
- `stack[]` — tecnologias mencionadas (core vs. exposição)
- `seniority` — senioridade declarada na vaga
- `salaryRange` — faixa salarial se visível; null se ausente
- `behaviors[]` — competências comportamentais citadas
- `archetype` — arquétipo detectado: "arquiteto técnico" / "evangelizador" / "engenheiro de produto" / "generalista" / "especialista" / "não identificado"

**Aplicar filtro de porte (D-26):**
- Se filtro = `médias+`: excluir vagas de `companySize: "pequena"` do array `jobs[]` final
- Se filtro = `grandes+`: incluir apenas `companySize: "grande"`
- Se filtro = `todas`: incluir todas independente do porte
- Vagas com `companySize: "desconhecido"` são INCLUÍDAS mesmo em filtros mais restritivos (benefício da dúvida)

- `commonChallenges[]` — desafios mais citados
- `archetypes[]` — arquétipos detectados com frequência
