# SKILL: Pesquisar Mercado

Coleta dados de vagas em portais BR públicos via WebSearch + WebFetch, extrai
e estrutura por vaga, gera sumário executivo e bloco `profileHints` pré-mastigado
para os 4 campos fixos do JobProfile, e persiste JSON em DATA_PATH/research/.

## Pre-Conditions

- DATA_PATH environment variable set and pointing to the data repository directory
- Node.js available (para gerar slug e salvar JSON via node -e)
- Acesso a WebSearch e WebFetch (portais públicos — sem autenticação)

## Execution Flow

### Step 1: Coletar Escopo da Pesquisa

Perguntar ao gestor (todas as perguntas em sequência; aceitar linguagem natural):

```
Cargo-alvo (ex: Engenheiro Sênior de Software, Arquiteto de Dados): ?

Localização (padrão: São Paulo, SP): [Enter para aceitar padrão]

Senioridade (Pleno / Sênior / Staff+): ?

Stacks-chave (ex: Java, Python, TypeScript): ?

Indústria/domínio (ex: EdTech, FinTech, varejo — opcional): [Enter para pular]

Profundidade:
  1. Enxuta  — 5-10 vagas, rápido (~5min)
  2. Média   — 15-25 vagas, equilibrado (~15min)
  3. Profunda — 30-50 vagas, abrangente (~30min+)
Escolha (1, 2 ou 3): ?
```

Confirmar escopo antes de prosseguir:
```
Pesquisando: {cargo} | {senioridade} | {stacks} | {localização}
Profundidade: {enxuta/média/profunda}
Prosseguir? (S/N)
```

### Step 2: Gerar Slug e Executar WebSearch nos Portais

Gerar slug a partir do escopo (apenas `[a-z0-9-]+`):
```bash
# Exemplo: senior-pd-java-python-ts-sp
# Regra: cargo-normalizado + stacks-principais + local-abreviado
# Sanitização: lowercase, espaços → hífens, remover acentos, remover caracteres especiais
```

Obter data atual:
```bash
node -e "console.log(new Date().toISOString().split('T')[0])"
```

Montar nome do arquivo: `{slug}-{YYYY-MM-DD}.json`

Verificar colisão (mesmo dia):
```bash
ls $DATA_PATH/research/{slug}-{YYYY-MM-DD}*.json 2>/dev/null
# Se existir, usar sufixo: {slug}-{YYYY-MM-DD}-2.json, -3.json, etc.
```

Executar WebSearch nos portais aprovados (usar portais e queries default de 05-01-PORTALS.md):

**Portais a pesquisar (na ordem de prioridade definida em 05-01):**
{INSERIR PORTAIS APROVADOS DE 05-01-PORTALS.md AQUI}

**Queries a usar:**
- Query PT: {query PT default de 05-01 para a senioridade escolhida, adaptada com stacks do gestor}
- Query EN: {query EN default de 05-01 para a senioridade escolhida, adaptada com stacks do gestor}

Para cada portal, registrar internamente:
```json
{ "portal": "{nome}", "query": "{query usada}", "url": "{url buscada}", "status": "ok|unavailable" }
```

Se WebSearch/WebFetch de um portal retornar 403/429/timeout → marcar `"status": "unavailable"` e prosseguir.

### Step 3: WebFetch das Descrições Mais Relevantes

A partir dos resultados de WebSearch, selecionar vagas mais relevantes para WebFetch completo:
- **Enxuta:** 5-10 descrições completas
- **Média:** 15-25 descrições completas
- **Profunda:** 30-50 descrições completas

Priorizar para WebFetch:
1. Vagas com título alinhado ao cargo-alvo
2. Empresas do setor relevante (se indústria foi especificada)
3. Vagas com faixa salarial visível no snippet (raro — priorizar quando aparecer)

Se WebFetch de uma descrição falhar → usar o snippet do WebSearch como fallback (marcar `fetchedFull: false`).

### Step 4: Extrair e Estruturar por Vaga

Para CADA vaga coletada, extrair:

```json
{
  "title": "Engenheiro Sênior Java",
  "company": "Empresa XYZ",
  "url": "https://...",
  "snippet": "Texto do snippet ou início da descrição",
  "portal": "gupy",
  "fetchedFull": true,
  "stack": ["Java", "Spring Boot", "Kubernetes"],
  "seniority": "Sênior",
  "salaryRange": "R$ 12k - 18k",
  "behaviors": ["Trabalho em equipe", "Comunicação clara"],
  "archetype": "arquiteto-tecnico"
}
```

**Notas de extração:**
- `stack`: distinguir core (exigido em >50% das vagas) vs. exposição (mencionado mas não obrigatório) — incluir ambos no array
- `seniority`: conforme declarado na vaga, não inferir se não explícito
- `salaryRange`: `null` se não visível ou não declarado na vaga
- `behaviors`: extrair dos campos de "perfil comportamental" ou "soft skills" da descrição
- `archetype`: detectar entre os tipos — `"evangelizador-poc-driver"`, `"arquiteto-tecnico"`, `"engenheiro-de-produto"`, `"generalista"`, `null` se não identificável

### Step 5: Gerar Summary Executivo e ProfileHints

Com base em TODAS as vagas extraídas no Step 4, gerar:

**Summary executivo:**
```json
{
  "commonTitles": ["títulos mais frequentes nas vagas encontradas"],
  "titleAliases": ["variações de título para o mesmo cargo"],
  "stackFrequency": { "Java": 18, "Python": 12, "TypeScript": 10, "Spring Boot": 15 },
  "emergingStack": ["tecnologias emergentes com frequência crescente mas ainda não dominante"],
  "salaryRange": "R$ 12k - 20k",
  "salarySource": "5 vagas com salário declarado",
  "commonBehaviors": ["comportamentos mais frequentes nas vagas"],
  "commonChallenges": ["desafios mais frequentes nas vagas"],
  "archetypes": ["arquiteto-tecnico", "evangelizador-poc-driver"],
  "trends": ["observações sobre o mercado nesta busca"],
  "redFlags": ["sinais de alerta comuns nas vagas (ex: ausência de menção a práticas ágeis)"]
}
```

**ProfileHints — tradutor para os 4 campos do JobProfile (D-01):**

IMPORTANTE: profileHints.qualifications usa o mesmo formato que JobProfile.qualifications — array de objetos `{ text, required }`, NÃO string[]. Responsabilidades, behaviors e challenges são `string[]`.

```json
{
  "responsibilities": [
    "Responsabilidade frequente nas vagas, escrita como bullet de perfil de cargo"
  ],
  "qualifications": [
    { "text": "Java 11+ com Spring Boot — presente em >70% das vagas", "required": true },
    { "text": "Experiência com Kubernetes ou equivalente", "required": false }
  ],
  "behaviors": [
    "Comportamento frequente nas vagas, escrito como competência comportamental"
  ],
  "challenges": [
    "Desafio frequente nas vagas, escrito como desafio do cargo"
  ],
  "suggestedTitle": "Engenheiro Sênior de Software",
  "suggestedExperienceLevel": "5-10 anos"
}
```

O system prompt a usar para gerar profileHints:
```
Você é especialista em recrutamento técnico para cargos de engenharia em P&D.
Com base nas vagas coletadas, gere o bloco profileHints para um JobProfile interno.
RESTRIÇÃO CRÍTICA: os 4 campos (responsibilities, qualifications, behaviors, challenges)
devem ser expressivos o suficiente para descrever o cargo, sem inventar campos extras.
Stack híbrido, arquétipo e nível de mercado devem ser expressos DENTRO desses campos.
Para qualifications, use formato { text, required } — required: true quando presente
em >50% das vagas, false caso contrário.
```

## Notes for Agent

- **Portais de 05-01:** A lista definitiva de portais e queries default está documentada em `.planning/phases/05-market-research-holistic-refinement/05-01-PORTALS.md`. Usar aqueles portais na ordem de prioridade definida.
- **profileHints.qualifications é ProfileItem[], não string[]:** Sempre usar `{ text, required }`. `required: true` para stacks/habilidades presentes em >50% das vagas; `false` para diferenciais.
- **Sem scraping agressivo:** Se um portal retornar erro HTTP, registrar como `"unavailable"` e prosseguir. Sem retentativas agressivas ou loops de retry. Uma tentativa por portal por sessão (D-03).
- **LinkedIn sem login:** Apenas cobertura via `site:linkedin.com/jobs` no Google WebSearch. Tentar WebFetch direto mas aceitar fallback para snippet se retornar bloqueio.
- **salaryRange null por padrão:** Apenas preencher se explicitamente declarado na vaga. Não inferir faixas a partir de seniority (D-08).
- **Sem dados pessoais:** Nunca extrair ou persistir nome de candidatos, currículos, ou qualquer informação pessoal identificável. Apenas dados das vagas/empresas (D-08).
- **node -e em vez de heredoc:** O heredoc shell tem problemas com aspas e newlines em JSON. O node -e preserva escape correto ao gravar o arquivo.
- **Colisão de slug no mesmo dia:** Verificar com `ls` antes de salvar. Sufixar com `-2`, `-3` se necessário (D-05).
- **Sanitização do slug:** Apenas `[a-z0-9-]+`. Remover acentos, converter espaços para hífens, remover caracteres especiais. Nunca incluir `..` ou `/` no nome do arquivo.

## Troubleshooting

**"DATA_PATH não está definido"**
→ Configurar a variável de ambiente:
```bash
export DATA_PATH=/caminho/para/repo-de-dados
```

**"Todos os portais retornaram unavailable"**
→ Verificar conexão com a internet. Testar manualmente: `curl -s "https://www.gupy.io" | head -20`
→ Se ambiente restrito, tentar via proxy ou aguardar e repetir.

**"Erro ao salvar o JSON"**
→ Verificar permissões: `ls -la $DATA_PATH/research/`
→ Verificar se DATA_PATH existe: `ls $DATA_PATH/`

**"profileHints está genérico demais"**
→ Aumentar a profundidade (média ou profunda) para coletar mais vagas antes de gerar.
→ Verificar se aiProfileInstructions da área está configurado no /settings — o /refinar-perfil pode complementar o que a pesquisa deixou genérico.
