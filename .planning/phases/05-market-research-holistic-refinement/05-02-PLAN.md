  <action>
Pesquisar títulos de cargos e faixas salariais do mercado BR de engenharia de software para criar `data/research/roles-map.json`.

**Fontes a consultar (em ordem de prioridade para dados salariais):**
1. Robert Half BR — Guia Salarial de TI (pesquisar via WebSearch "robert half guia salarial 2026 TI engenharia software")
2. Glassdoor BR — pesquisar salários por cargo (via WebFetch ou WebSearch "glassdoor salario engenheiro software sênior são paulo 2026")
3. Catho — pesquisa salarial (via WebSearch "catho pesquisa salarial 2026 desenvolvedor sênior")
4. Revelo — página de salary (via WebSearch "revelo salary software engineer brazil 2026")
5. Infojobs — pesquisa de salários (best-effort)

**Cargos obrigatórios a mapear** (mínimo para o arquivo ser útil):

Para engenharia de software geral:
- Desenvolvedor Júnior / Junior Software Engineer
- Desenvolvedor Pleno / Software Engineer
- Engenheiro Sênior de Software / Senior Software Engineer
- Tech Lead / Líder Técnico
- Staff Engineer
- Principal Engineer
- Arquiteto de Soluções / Solutions Architect
- Engenheiro de Software Especialista

Para P&D/IA (relevante para o contexto Lyceum):
- Engenheiro de IA / AI Engineer
- Engenheiro de Machine Learning / ML Engineer
- Engenheiro de Dados Sênior / Senior Data Engineer

**Aliases a documentar por cargo:**
Registrar variações em PT-BR e EN que os portais usam para o mesmo papel.

**Faixas salariais:**
- Quando encontrar faixa pública, registrar com min/max/currency/location/salarySource
- Quando não encontrar, registrar `"salaryRange": null`
- NÃO inventar valores — dados reais ou null

**Tendências emergentes:**
- Verificar quais títulos estão surgindo (ex: "AI Engineer", "Platform Engineer", "Developer Advocate")
- Verificar quais estão se tornando obsoletos
- Documentar em `notes` de cada entry relevante

**Criação do arquivo:**
1. Verificar se `data/research/` existe; se não, criar:
   ```bash
   mkdir -p $DATA_PATH/research
   ```
   (ou usar o path direto do repositório se DATA_PATH não estiver definido no ambiente do executor)

2. Salvar o arquivo via node -e (NÃO heredoc — evita problemas com aspas e caracteres especiais):
   ```bash
   node -e "
   const fs = require('fs');
   const path = require('path');
   const dataPath = process.env.DATA_PATH || './data';
   const dir = path.join(dataPath, 'research');
   fs.mkdirSync(dir, { recursive: true });
   const rolesMap = {
     updatedAt: new Date().toISOString(),
     source: 'pesquisa manual via WebSearch/WebFetch em Robert Half BR, Glassdoor BR, Catho, Revelo',
     roles: [ /* array completo aqui */ ]
   };
   fs.writeFileSync(path.join(dir, 'roles-map.json'), JSON.stringify(rolesMap, null, 2));
   console.log('roles-map.json criado em', dir);
   "
   ```

**Mínimo aceitável:** 10 entries de cargos com aliases, pelo menos 5 com salaryRange não-null.
